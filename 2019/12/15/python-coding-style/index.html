python编程风格指南

本文根据python官方指南PEP8和google的python style guides，总结了一份python的风格指南。
要注意的是，这份指南只是一份指导，而代码的可读性才是真正重要的东西！

空行

函数内部用空行分割不同的逻辑块。

文件编码

字符串引号

单引号或双引号都可以，只要保持风格一致即可。
另外，避免使用\来转义引号，应该使用另一种引号将其包围。

    注释

    如果没有注释，这些代码的可读性会更差的情况下，应当加上注释（换句话说，代码的可读性更重要，如果代码很简洁，注释是没有必要的）。

注释应当是一句完整的句子，首字母应该大写（除非是变量，绝不要改变变量的大小写）。

非英语母语的开发者，也请用英文写注释。除非你能120%地确定这份代码不会被其它国家的人阅读。

命名规范

  • 避免l（会与阿拉伯数字1搞混）、O、o（会与0搞混）这样的变量名。
  • 包的命名：都是小写字符，应当尽可能地简短，尽可能地不使用下划线。
  • 模块的命名：都是小写字符，应当尽可能地简短，可以通过下划线增强可读性。当python用于封装c/c++c++，模块名应该以下划线开头，如_socket。
  • 类的命名：应该遵循驼峰命名法。
  • 自定义异常：因为自定义异常也是一个类，所以也应该遵循驼峰命名法，另外，还应该以Error作为结尾。
  • 变量和函数：使用下划线命名法，全部小写。
  • 如果命名和python关键词冲突，后面加一个_，如class_。
  • 在类的私有变量或成员函数前面加一个_，如_var。
  • const变量全部大写，用下划线分割，如MAX_OVERFLOW
